> 最后更新: 2026-05-26

## 一、核心概念：DNS 记录类型互相独立

DNS 中不同记录类型各司其职，互不干扰：

| 记录类型 | 作用 | 类比 |
|---------|------|------|
| **A / AAAA** | 域名 → IP 地址（网站访问） | 门牌号 |
| **CNAME** | 域名 → 另一个域名（别名转发） | 门牌别名 |
| **MX** | 域名 → 邮件服务器（邮件投递） | 信箱编号 |
| **TXT / SPF** | 域名验证、反垃圾邮件 | 信箱认证标签 |

**关键结论**：给 `eforge.xyz` 添加 MX 记录，**完全不影响**它的 A 记录（网站访问）。
就像给房子加一个信箱，不会改变门牌号。

## 三、邮件路由架构（变更前后对比）

### 3.1 变更前：`@mail.eforge.xyz`

```
用户写信: abc@mail.eforge.xyz  (23 字符，偏长)
    ↓
DNS 查询 mail.eforge.xyz 的 MX 记录
    ↓
Cloudflare Email Routing
    ↓
转发给 Vmail Worker (mail.eforge.xyz)
    ↓
Worker 中 EMAIL_DOMAIN = "mail.eforge.xyz"
```

**问题**：邮箱后缀 `@mail.eforge.xyz` 共 18 字符，`mail.` 是冗余前缀。

### 3.2 变更后：`@eforge.xyz`

```
用户写信: abc@eforge.xyz  (15 字符，缩短 35%)
    ↓
DNS 查询 eforge.xyz 的 MX 记录
    ↓
Cloudflare Email Routing
    ↓
转发给 Vmail Worker (mail.eforge.xyz)  ← Worker 访问地址不变
    ↓
Worker 中 EMAIL_DOMAIN = "eforge.xyz"  ← 仅这里变了
```

**优势**：
- 邮箱后缀缩短 35%（18 → 10 字符）
- 更专业：`user@eforge.xyz` 而非 `user@mail.eforge.xyz`
- 与 Gmail、Outlook 等主流邮箱风格一致（直接用根域）

## 四、关键分离：邮件域名 vs Worker 访问域名

这是最容易混淆的地方，必须理解清楚：

```
┌─────────────────────────────────────────────────────┐
│  邮件域名 (EMAIL_DOMAIN)                              │
│  = 用户收发的邮箱后缀                                  │
│  = abc@eforge.xyz 中的 "eforge.xyz"                   │
│  = Cloudflare Email Routing 匹配的域名                │
├─────────────────────────────────────────────────────┤
│  Worker 访问域名                                       │
│  = 用户浏览器访问邮箱界面的地址                         │
│  = https://mail.eforge.xyz                            │
│  = Cloudflare Worker 绑定的自定义域名                  │
└─────────────────────────────────────────────────────┘

两者完全独立！可以不同！
```

**实际映射**：

| 概念 | 值 | 在哪里配置 |
|------|-----|-----------|
| 邮件域名（邮箱后缀） | `eforge.xyz` | Worker 环境变量 `EMAIL_DOMAIN` |
| Worker 访问域名（邮箱界面） | `mail.eforge.xyz` | Cloudflare Worker 自定义域名绑定 |
| Email Routing 转发目标 | Worker URL 重写 | Cloudflare Dashboard → Email → Routing |

#### 步骤 1：添加 MX 记录

在 Cloudflare Dashboard → `eforge.xyz` → DNS → 添加记录：

```
类型:    MX
名称:    @
优先级:  10
目标:    mail.eforge.xyz
```

或者使用 Cloudflare Email Routing 的专用 MX：
```
类型:    MX
名称:    @
优先级:  10
目标:    mx1.cloudflaremail.com (优先级 10)
         mx2.cloudflaremail.com (优先级 20)
         mx3.cloudflaremail.com (优先级 30)
```

#### 步骤 2：配置 Email Routing

1. Cloudflare Dashboard → `eforge.xyz` → Email → Email Routing
2. 启用 Email Routing
3. 添加路由规则：
   - 匹配：`*@eforge.xyz`
   - 动作：Worker URL 重写
   - 目标：`https://mail.eforge.xyz/api/v1/email/receive`（或按实际 API 路径）

#### 步骤 3：更新 GitHub Secrets

在 GitHub 仓库 Settings → Secrets and variables → Actions：

| Secret | 旧值 | 新值 |
|--------|------|------|
| `EMAIL_DOMAIN` | `mail.eforge.xyz` | `eforge.xyz` |

更新后，下一次 `git push` 到 `main` 分支会自动部署新配置。

## 七、为什么其他子域名不受影响

### 7.1 DNS 查询是精确匹配的

```
查询 eforge.xyz 的 A 记录    → 返回 198.18.0.123  (网站)
查询 eforge.xyz 的 MX 记录    → 返回 mail.eforge.xyz (邮件)
查询 ai.eforge.xyz 的 A 记录  → 返回 198.18.0.145  (AI 服务)
查询 mail.eforge.xyz 的 CNAME → 返回 Workers 地址    (邮箱界面)
```

每个查询独立处理，互不干扰。

### 7.2 MX 记录只影响邮件发送方

MX 记录的作用：**告诉世界"发给 @eforge.xyz 的邮件送到哪里"**。

- 浏览器访问 `https://eforge.xyz` → 查 A 记录 → 不受 MX 影响
- 浏览器访问 `https://ai.eforge.xyz` → 查 A 记录 → 不受 MX 影响
- 浏览器访问 `https://mail.eforge.xyz` → 查 CNAME → 不受 MX 影响
- 只有**发邮件给** `xxx@eforge.xyz` 时，才会查 MX 记录

### 7.3 子域名的 MX 继承

```
发给 abc@eforge.xyz     → 查 eforge.xyz 的 MX → 找到 → 投递 ✅
发给 abc@ai.eforge.xyz  → 查 ai.eforge.xyz 的 MX → 无 → 查父域 eforge.xyz 的 MX → 继承 ✅
发给 abc@mail.eforge.xyz → 查 mail.eforge.xyz 的 MX → 无 → 查父域 eforge.xyz 的 MX → 继承 ✅
```

如果将来需要 `@ai.eforge.xyz` 也作为邮箱后缀，只需：
1. 在 Worker 的 `EMAIL_DOMAIN` 中添加：`"eforge.xyz,ai.eforge.xyz"`
2. 在 Email Routing 中添加对应路由规则

## 八、多域名支持

Vmail Worker 原生支持多域名（逗号分隔）：

```toml
# wrangler.toml
EMAIL_DOMAIN = "eforge.xyz,ai.eforge.xyz"
```

代码逻辑（`worker/src/index.ts:329`）：
```typescript
const emailDomain = c.env.EMAIL_DOMAIN
  .split(',')
  .map(d => d.trim());  // → ["eforge.xyz", "ai.eforge.xyz"]
```

前端 `/config` 接口返回域名数组，注册页面自动展示所有可用后缀。

## 九、故障排查

### 9.1 邮件收不到

| 症状 | 可能原因 | 检查方法 |
|------|---------|---------|
| 发信方收到退信 | MX 记录未正确设置 | `dig eforge.xyz MX` |
| Cloudflare 收到但 Worker 没收到 | Email Routing 规则未配置 | Dashboard → Email → Routing |
| Worker 收到但提示域名不匹配 | `EMAIL_DOMAIN` 未更新 | `curl https://mail.eforge.xyz/config` |

### 9.2 网站访问异常

| 症状 | 可能原因 | 检查方法 |
|------|---------|---------|
| eforge.xyz 打不开 | A 记录被误删/修改 | `dig eforge.xyz A` |
| mail.eforge.xyz 打不开 | CNAME 被误删 | `dig mail.eforge.xyz CNAME` |
| ai.eforge.xyz 打不开 | A 记录被误删 | `dig ai.eforge.xyz A` |

> **注意**：添加 MX 记录**不会**导致以上任何问题。如果网站异常，检查是否误操作了 A/CNAME 记录。

|------|-----|------|
| 邮箱后缀 | `@eforge.xyz` | 简洁专业，缩短 35% |
